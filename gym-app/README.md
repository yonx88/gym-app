# 🏋️ نادي — تطبيق متابعة تماريني

تطبيق ويب يشتغل على الجوال (Mobile-First) بالعربي: تتابع تمارينك اليومية،
الأوزان والجولات، صور التمارين، السعرات، وجدول حضورك الأسبوعي.

- **الواجهة:** Angular 17
- **السيرفر:** Node.js + Express
- **قاعدة البيانات:** SQLite **المدمجة في Node** (بدون أي تنصيب أو تجميع)
- **النشر:** خدمة واحدة (السيرفر يقدّم الواجهة) → رابط واحد للجميع

---

## ✨ المميزات

- تسجيل دخول بالإيميل وكلمة المرور (مشفّرة + جلسة JWT).
- جداول جاهزة: **بش بل ليق (PPL)** · **علوي/سفلي** · **كل يوم عضلة**، أو **تبني جدولك بنفسك**.
- تحدد أيام النادي، عضلة كل يوم، والكارديو (الجهاز والمدة)، وطولك ووزنك.
- شاشة تمارين اليوم: صورة التمرين + تسجيل الوزن والجولات، **ويعرض آخر وزن شلته**.
- تشيك ليست ✓ لكل تمرين + زر "أنهيت اليوم" + عدّاد سعرات فوق.
- تايم شيت أسبوعي يبيّن الأيام اللي جيتها واللي فاتتك.

---

## ⚙️ المتطلبات

- **Node.js نسخة 22.5 أو أحدث** (القاعدة تستخدم SQLite المدمجة في Node).
  تأكد: `node --version`

## 💻 التشغيل المحلي (على جهازك)

تحتاج تيرمناليْن:

**1) السيرفر:**
```bash
cd backend
npm install
npm start            # يشتغل على http://localhost:3000
```

**2) الواجهة (تيرمنال ثاني):**
```bash
cd frontend
npm install
npm start            # يفتح http://localhost:4200
```

الواجهة تتصل بالسيرفر تلقائياً عبر بروكسي التطوير (proxy.conf.json).
افتح `http://localhost:4200` وسجّل حساب جديد.

> رسالة `ExperimentalWarning: SQLite ...` في ترمنال السيرفر طبيعية، تجاهلها.

## 🌍 النشر على الإنترنت (Railway) — رابط واحد للجميع

عند النشر، السيرفر يقدّم الواجهة بنفسه، فيصير **خدمة واحدة ورابط واحد**.

### الخطوة 1: ارفع المشروع على GitHub
أنشئ مستودع (repo) جديد وارفع فيه مجلد المشروع كامل.
(ملف `.gitignore` يستبعد `node_modules` والملفات المؤقتة تلقائياً.)

### الخطوة 2: أنشئ المشروع في Railway
1. ادخل [railway.com](https://railway.com) وسجّل دخول بحساب GitHub.
2. **New Project → Deploy from GitHub repo** واختر المستودع.
3. Railway بيكتشف Node ويشغّل تلقائياً `npm run build` ثم `npm start`.
   (نسخة Node مثبّتة على 22 عبر ملف `.nvmrc`.)

### الخطوة 3: أضف قرص دائم (Volume) — مهم عشان البيانات ما تنمسح
داخل الخدمة: **Variables/Settings → Volumes → Add Volume**
واربطه (Mount path) على:
```
/data
```

### الخطوة 4: أضف متغيّرات البيئة
في **Variables** أضف:

| المتغيّر       | القيمة                                   |
|----------------|------------------------------------------|
| `DATA_DIR`     | `/data/db`                               |
| `UPLOADS_DIR`  | `/data/uploads`                          |
| `JWT_SECRET`   | نص طويل وعشوائي (سرّك الخاص — لا تشاركه)  |

> `PORT` تضبطه Railway تلقائياً، لا تحتاج تضيفه.

### الخطوة 5: افتح الرابط وشاركه
**Settings → Networking → Generate Domain** بيعطيك رابط مثل
`https://اسم-مشروعك.up.railway.app` — هذا الرابط يفتحه أي شخص ويسجّل ويستخدم التطبيق.

> أول بناء ياخذ بضع دقائق. أي تحديث ترفعه على GitHub (push) ينشر تلقائياً،
> وبياناتك تبقى محفوظة في الـ Volume.
